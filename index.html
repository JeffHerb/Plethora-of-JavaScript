JavaScript Libraries:

a collection of highly abstracted javascript functions.

Provide developers with:

  • Reusable functions
  • Little or No setup headaches
  • Flexibility in coding structure and interoperability*

JS Libraries: Flexibility in coding

Types of Libraries

  • DOM Manipulation
  • Widget/GUI Components

JS Libraries - DOM Manipulation

Purpose: Find specific element in the DOM, Bind event to elements or adding/removing elements to/from the DOM.

  • jQuery
  • Zeptos
  • Dojo

  • How generic or opinionated are they.
  • Their syntax/learning curve
  • Plugins/Extendable

JS Libraries - Widget/GUI Components

Purpose: Add pre-built components to any site or application with minumal markup on the developer side.

  • jQuery UI
  • Dojo toolkit
  • Ext JS

  $( "#datepicker" ).datepicker();

Widget/GUI Components: Considerations

  • DOM Manipulation ties!

AngularJS is one of the oldest of the frameworks

JS Framework: AngularJS Pro

  • Two-way Data Binding
  • Test Driven Development
  • 3rd-party libraries built for AngularJS

JS Framework: AngularJS Con

  • IE8+
  • Not for constant DOM Manipulation
  • Somewhat of a learning curve

JS Framework: Backbone

Known as the super small and the Professional framework

JS Framework: Backbone

The Non-opinionated framework

If you ignore Underscore.js

JS Framework: Backbone Pro

  • Smallest in framework size 6.5kb
  • Pure MVC framework
  • Maturity from simplicity

JS Framework: Backbone Con

  • No opinions, you bring everthing you want
  • More initial setup
  • Requires a RESTful Backend

JS Framework: Ember

The new guy on the block; but gaining in popularity.

JS Framework: Ember

Ember is the opinionated framework

Convention over configuration!

JS Framework: Ember Pro

  • Great for larger application
  • Lots of pre-built features and componet level design
  • 100% Comminity Supported

JS Framework: Ember Con

  • Huge footprint at 71kb
  • Very Opinionated
  • Overkill for smaller apps/pages

JS Frameworks: Things to consider

  • The size of your project and its complexity
  • The libraries you want to work with
  • The amount of data
